READ ME
Antoine Lohézic

Toutes les travaux réalisés ci-dessous ont été réalisés dans le but d'avoir
une bonne note.

1. La première étape consiste à forker le dépôt avec les consignes sur mon
compte github. Cela ce fait simplement, à la main, en utilisant le bouton
'Fork' du dépôt de départ. Tous les élements se trouvent alors sur mon compte,
notamment les consignes.

2. La deuxième étape est le clonage de mon dépôt en local. 

3. Ensuite, j'ai écrit le script. Il s'éxecute avec la commande 'bash
fich_semaine_derniere.sh'. Le script accueille l'utilisateur avec son nom et
la date. Il affiche ensuite l'ensemble des éléments présents dans la 'home'.
On peut alors choisir le dossier à explorer. Le script affiche le contenu du
dossier demandé par l'utilisateur.

4. La quatrième étape regroupe toutes les étapes ('git add', 'git commit') nécessaires jusqu'à pousser
le fichier sur le dépôt distant.

5. Ensuite, j'ai créé une nouvelle branche.

6. Lors de la sixième étape, j'ai ajouté une commande au script précédent.
Elle permet d'afficher, dans le dossier choisi par l'utilisateur, les
fichiers modifiés lors des sept derniers jours.

7. Ensuite, il s'agit de pousser le script modifié vers le dépôt distant, de la même façon que
précédement.

8. La huitième étape consiste à comparer et fusionner les deux branches
précédement poussées sur le dépôt distant.

9. Ensuite, j'ai fusionné les deux branches en local.

10. Les consignes ne sont plus utiles car j'ai fini le projet. Je le supprime
donc et pousse les modifications vers le dépôt distant.
